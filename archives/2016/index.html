Ubuntu下反编译apk

工具下载

  1. apktool：这个工具用于最大幅度地还原APK文件中的9-patch图片、布局、字符串等等一系列的资源。
    下载地址：http://ibotpeaches.github.io/Apktool/install/

下载最新版本，例如我下载的是apktool_2.0.3.jar，下载后重命名为apktool.jar

  1. dex2jar：这个工具用于将dex文件转换为jar文件
    下载地址：http://sourceforge.net/projects/dex2jar/files/
  1. jd-gui：这个工具用于将jar文件转换为java代码
    下载地址：http://jd.benow.ca/

反编译代码

1.反编译资源文件
首先准备一个apk文件，例如demo.apk，然后重命名为demo.zip，然后解压，你会发现有资源文件，但是这些资源文件都是在打包的时候被编译过了，所以直接打开的话是看不到明文的。要想将apk文件中的资源反编译出来，需要用到apktool工具。
将之前下载的apktool.jar文件和apk文件放在同一目录，然后执行命令：

java -jar apktool.jar d demo.apk

其中d是decode的意思，表示我们要对Demo.apk这个文件进行解码。那除了这个基本用法之外，我们还可以再加上一些附加参数来控制decode的更多行为：

-f 如果目标文件夹已存在，则强制删除现有文件夹（默认如果目标文件夹已存在，则解码失败）。
-o 指定解码目标文件夹的名称（默认使用APK文件的名字来命名目标文件夹）。
-s 不反编译dex文件，也就是说classes.dex文件会被保留（默认会将dex文件解码成smali文件）。
-r 不反编译资源文件，也就是说resources.arsc文件会被保留（默认会将resources.arsc解码成具体的资源文件）。

运行成功后会在目录下生成一个文件夹，里面就是反编译apk后的资源文件。

2.反编译代码
首先，先解压下载好的dex2jar，解压后我们需要用到d2j-dex2jar.sh这个文件，将demo.zip解压，里面有个classes.dex文件，这个classes.dex文件就是存放所有java代码的地方了。将它拷贝到dex2jar解压后的目录下，并执行命令：

d2j-dex2jar.sh classes.dex

如果命令报错：

d2j-dex2jar.sh: command not found

那么执行命令：

sh d2j-dex2jar.sh /apk/classes.dex

如果命令报错：

./d2j_invoke.sh: Permission denied

那么再执行：

sudo chmod +x ./d2j_invoke.sh

然后再执行：

sh d2j-dex2jar.sh classes.dex

执行成功后会发现生成了一个classes-dex2jar.jar文件，这个文件就是java代码的jar文件。
最后，通过jd-gui工具打开这个jar文件，就可以阅读转换出来的java代码啦。

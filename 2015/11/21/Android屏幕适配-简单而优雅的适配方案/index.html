Android屏幕适配:简单而优雅的适配方案

  • android
学习

做Android开发，为了让开发出的应用能够美观的显示在不同尺寸、分辨率的设备上，免不了要做屏幕适配。首先，了解下一些最基本的相关概念。

基本概念

1，屏幕分辨率
屏幕分辨率是指在横纵向上的像素点数，单位是px， 例如我用的测试机是1920乘1080，代表纵向1920px, 横向1080px
2，英寸
屏幕的物理尺寸，例如手机屏幕大小5.5英寸指的是屏幕的对角线长度。1英寸=2.45厘米。
3，屏幕像素密度
指每英寸上的像素点数，单位是dpi，屏幕像素密度与屏幕尺寸和屏幕分辨率有关。例如320乘480分辨率的手机，宽2英寸，高3英寸，每英寸包含的像素点
的数量为320/2=160dpi(横向)或480/3=160dip(纵向)，那么160就是这部手机的dpi。(因为大部分手机屏幕使用正方形的像素点，所以横向和纵向的这个值相等)
4，设备独立像素
使用dp或者dip表示，在屏幕密度dpi=160的屏幕上，1dp=1px
5，屏幕密度
使用density表示，density = dpi/160。

Android中常见的dpi有如下四种：

mdpi (dpi = 160, density = 1)
hdpi (dpi = 240, density = 1.5)
xhdpi (dpi = 320, density = 2)
xxhdpi (dpi = 480, density = 3)

由于各种屏幕的像素密度都有所不同，因此相同数量的像素在不同设备上的实际大小也有所差异，这样使用像素定义布局尺寸就会产生问题。应该采用谷歌推荐的dip或sp单位制定尺寸。虽然dip可以解决不同像素密度的问题，使得1dip在不同像素密度上面的显示效果相同，但是由于Android设备屏幕宽度并不都是相同，虽然结合match_parent, wrap_content,weight等能够实现大部分适配，但这还不是完美的解决方案。下面，让我来介绍一种简单而优雅的比较完美的适配方案。

适配方案

使用320px=320dp的分辨率为基准，将屏幕宽度分为320等分。那么，在320px分辨率的手机下，1单位长度=1px，在640px分辨率下，1单位长度=2px。先获取用户手机的屏幕宽度，最后算出设置的px在用户手机上真实的px值。
下面这个类是核心，通过这个类，我们可以在布局文件中设置控件的宽和高，宽和高都是以dip为单位，因为将屏幕分为320等分，所以设置的宽最大为320dip.

/**
 * 实现分辨率自适应的、用于替换系统的Resources
 * Created by Daniel_Wen on 2015/11/21.
 */
public class CompatResources extends Resources{
    private static final String TAG = "CompatResources";

    private static CompatResources mCompatResources;
    private Resources mOriginalResources;

    private CompatResources(Resources originalResources){
        super(originalResources.getAssets(), originalResources.getDisplayMetrics(), originalResources.getConfiguration());
        mOriginalResources = originalResources;
        updateConfiguration();
    }

    private void updateConfiguration(){
        DisplayMetrics currentDisplayMetrics = getDisplayMetrics();
        currentDisplayMetrics.density = DimenUtil.getScreenWidthRate(mOriginalResources);
        currentDisplayMetrics.scaledDensity = currentDisplayMetrics.density;
        currentDisplayMetrics.densityDpi = (int)(currentDisplayMetrics.density * DisplayMetrics.DENSITY_DEFAULT);
        currentDisplayMetrics.xdpi = currentDisplayMetrics.densityDpi;
        currentDisplayMetrics.ydpi = currentDisplayMetrics.densityDpi;

        try {
            Class<?> resourcesClass = Class.forName("android.content.res.Resources");
            Field[] fields = resourcesClass.getDeclaredFields();
            if(fields != null && fields.length > 0){
                for(Field field : fields){
                    if("mTmpMetrics".equals(field.getName())){
                        field.setAccessible(true);
                        field.set(this, currentDisplayMetrics);
                    }
                }
            }
        } catch (ClassNotFoundException e) {
            Log.e(TAG, e.getMessage(), e);
        } catch (IllegalAccessException e) {
            Log.e(TAG, e.getMessage(), e);
        } catch (Exception e){
            Log.e(TAG, e.getMessage(), e);
        }
    }

    public Resources getOriginalResources(){
        return mOriginalResources;
    }

    public static CompatResources getCompatResources(Resources originalResources){
        if(mCompatResources == null){
            mCompatResources = new CompatResources(originalResources);
        }else{
            Resources oldOriginalResources = mCompatResources.getOriginalResources();
            if(!oldOriginalResources.equals(originalResources)
                    || !oldOriginalResources.getDisplayMetrics().equals((Object)originalResources.getDisplayMetrics())){
                mCompatResources = new CompatResources(originalResources);
            }
        }
        return mCompatResources;
    }
}

我们获取系统的Resources是通过getResources()方法，现在我们需要获取上面这个实现了适配的CompatResources。所以需要进行如下操作：1，写一个BaseActivity，在这个BaseActivity中重写getResources方法来获取CompatResources，以后自己写的Activity都继承BaseActivity。2，写一个BaseApplication，并重写getResources方法，

public class BaseActivity extends AppCompatActivity{

    protected final AppCompatActivity mContext = this;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
    }

    @Override
    public Resources getResources() {
        return CompatResources.getCompatResources(super.getResources());
    }

    protected void startIntent(Class<?> clazz){
        Intent intent = new Intent(mContext,clazz);
        intent.setAction(clazz.getSimpleName());
        startActivity(intent);
    }
}

public class BaseApplication extends Application{

    public static Context context;

    public BaseApplication(){
        super();
        context = BaseApplication.this;
    }

    @Override
    public void onCreate() {
        super.onCreate();
    }

    @Override
    public Resources getResources() {
        return CompatResources.getCompatResources(super.getResources());
    }
}

除了在布局文件中对控件进行设置宽高，我们还会在代码中进行设置，下面这个工具方法实现了适配。使用方式：例如设计师给了一个在640的分辨率下的设计稿，其中有个控件设计的宽和高为64px*64px，那么我们在布局中设置值得时候就是宽高分别为32dip, 在代码中就使用DimenUtil.getRealPx(context, 32)来获取真实的px值。

/**
 * 用于处理多分辨率下不同尺度的问题。
 * 以320×480分辨率为准，该分辨率下1基准px=1px.
 * 其他分辨率下各种基准分辨率适配
 * Created by Daniel_Wen on 2015/11/21.
 */
public class DimenUtil {
    private static final String TAG = "DimenUtil";

    private static DisplayMetrics mDisplayMetrics;
    private static Configuration mConfiguration;

    /**
     * 以用户手机屏幕的宽为基准，算出设置的px在用户手机上真实的px值
     * @param context
     * @param px 设置的在320*480分辨率下的基准px值
     * @return 经过换算后的真实px
     * */
    public static int getRealPx(Context context, float px){
        //兼容WRAP_CONTENT, MATCH_PARENT(FILL_PARENT)
        if(px == 0 || px == ViewGroup.LayoutParams.WRAP_CONTENT || px == ViewGroup.LayoutParams.MATCH_PARENT){
            return (int)px;
        }else{
            float realWidth = getScreenWidthRate(context.getResources()) * px;
            Log.e(TAG, "real width:" + realWidth);
            return (int)realWidth;
        }
    }

    /**
     * 将一个屏幕宽度的总像素数平均分成320份，每一份对应具体的像素就可以了
     */
    public static float getScreenWidthRate(Resources resources){
        fillDisplayMetrics(resources);
        int width = isScreenOrientationPortrait(resources)?mDisplayMetrics.widthPixels:mDisplayMetrics.heightPixels;
        Log.e(TAG, "width:" + width);
        return width/320f;
    }

    private static void fillDisplayMetrics(Resources resources){
        if(mDisplayMetrics == null){
            if(resources instanceof CompatResources){
                mDisplayMetrics = ((CompatResources) resources).getOriginalResources().getDisplayMetrics();
            }else{
                mDisplayMetrics = resources.getDisplayMetrics();
            }
        }
    }

    /**
     * 判断当前屏幕是否为竖屏
     * */
    private static boolean isScreenOrientationPortrait(Resources resources){
        if(mConfiguration == null){
            if(resources instanceof CompatResources){
                mConfiguration = ((CompatResources) resources).getOriginalResources().getConfiguration();
            }else{
                mConfiguration = resources.getConfiguration();
            }
        }
        return mConfiguration.orientation == Configuration.ORIENTATION_PORTRAIT;
    }
}

通过这种方式可以很好的实现屏幕适配，有木有觉得非常优雅~~

使用Github和Hexo搭建个人博客

  • Hexo
  • 搭建博客
学习

一直想搭建个个人博客，看到很多人使用Hexo，感觉挺不错的，心动不如行动，那就开始吧！

1，安装Node.js

到Node.js官网https://nodejs.org/download/下载对应版本，安装很简单，一直next就行啦。

2，安装Git

我的电脑是windows系统，到http://msysgit.github.io/上下载msysgit并进行安装

3，安装Hexo

Node和Git都安装好后，打开cmd命令提示符，然后可以执行如下命令安装hexo：

npm install -g hexo

然后初始化Hexo，首先得先打开git bash，进入任意磁盘，假如是F盘，然后依次执行如下命令:

hexo init blog
cd blog
npm install

这样，就会在F:\blog目录下安装node_modules，然后再执行如下命令：

hexo server

然后在浏览器中打开：http://localhost:4000/，这时就可以看到Hexo已为你生成了一篇blog。可以按Ctrl+C停止server.

4，新建文章

在git bash命令行中cd 到F:\blog下，然后执行如下命令：

hexo new “Hexo搭建博客”

这样就会在F:\blog\source_posts目录下生成一个”Hexo搭建博客.md”文件，并且刷新http://localhost:4000/，可以发现多了一篇新的文章“Hexo搭建博客”.再执行如下命令，将markdown文件生成静态网页：

hexo generate

该命令执行完后，会在F:\blog\public\目录下生成一系列html,css等文件

5，编辑文章

因为生成的是markdown文件，可以使用markdown编辑器对”Hexo搭建博客.md”文件进行编辑，markdown的语法可参考：http://wowubuntu.com/markdown/,打开该文件发现已生成如下内容：

title: Hexo搭建博客
date: 2015-04-10 13:13:24
categories:

tags:

“categories”表示把文章放在哪个分类下，”tags”表示为该文章添加标签。修改上面的内容成如下：

title: Hexo搭建博客
date: 2015-04-10 13:13:24
categories: Hexo

tags: [Hexo 搭建博客]

表示该文章将放在Hexo分类下，并且添加了两个标签：Hexo、搭建博客。标签使用[]括起来，标签之间使用“，”隔开

6，部署到Github上

首先注册github账号，然后new一个repository,比如我的github账号是helloworld1992，那么创建的repository的名字为helloworld1992.github.io，然后编辑_config.yml文件（在F:\blog下），

deploy:
type: github
repository: https://github.com/helloworld1992/helloworld1992.github.io.git
branch: master

然后在blog目录下执行如下git bash命令：

hexo deploy

如果提示需要输入用户名和密码，输入即可
成功后浏览器输入http://helloworld1992.github.io/就可以看到自己的博客啦

7，更换主题

Hexo提供了很多主题，可到这里https://github.com/hexojs/hexo/wiki/Themes#选择自己喜欢的主题，我选择的是yilia，安装主题方法如下：

  1. 将git bash 切到F:\blog目录下，然后执行下面的命令：

    git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

  2. 修改F:\blog目录下的_config.yml中的theme为yilia

  3. 执行命令：

    cd themes/yilia
    git pull

好啦，现在重新打开浏览器浏览会发现主题变啦！

8，Github的域名绑定

如何购买域名就不说了，可以去网上搜教程（我是在GoDaddy上买的，支持支付宝付款），GoDaddy的NameServer有时会被墙，因此强烈推荐国内的DNSPod解析
域名，免费服务真心不错。首先进入DNSPod,然后添加购买的域名，点击进去就像如下图所示：

图中的NS记录是DNSPod默认的绑定，其中的记录值配置到GoDaddy的Nameservers中即可.其中的CNAME记录类型是我自己添加的，后面的记录值就是之前在github上建的仓库helloworld1992.github.io，CNAME用于绑定一个域名，先新建CNAME文件，我是放在/blog/source/ 目录下，然后把域名写进这个文件，例如我的域名是belief1992.com，则直接把belief1992.com写进CNAME文件，然后执行如下命令：

hexo clean
hexo generate
hexo deploy

这样，打开github进入仓库，就会发现根目录下多了一个CNAME文件。
好啦，大功告成啦，打开浏览器，输入“belief1992.com”访问，是不是很酷啊~~
